Class: Message

Message(type, payload, metadataopt, idopt, timestampopt)

Base class for message objects exchanged over WebSockets or HTTP.

Represents a serializable message with a type, payload, id, timestamp, and optional metadata. The type is used to route the message to the correct handler/event.

Subclasses should define domain-specific message types (e.g., Info, Call, Result).

FROM_JSON :Map.<string, function()>

Global registry for deserialization functions keyed by message type.

Subclasses should register their static fromJSON() function like:

Message.FROM_JSON.set('ClientInfo', ClientInfo.fromJSON);

This registry enables dynamic dispatch of deserialization logic without introducing circular imports.

(static) fromJSON(json) → {Message}

Reconstructs a Message or a subclass instance from a plain JSON object.

Dispatches based on the type field and delegates to the registered subclass’s fromJSON() method when available. This enables correct rehydration of complex structures (e.g., Sets, Maps, typed instances).

If no deserializer is registered for the given type, a generic Message instance is returned as a fallback.

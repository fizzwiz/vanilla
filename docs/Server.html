Class: Server

Server(info, netInfoUrl, httpServer, jwtSecret, jwtExpiryopt)

A Node that can both request outbound WebSocket connections and accept inbound connections.

Behavior

  • Other nodes (e.g., Clients) connect to this Server by discovering its address via NetInfo.
  • It communicates over WebSockets and does not handle HTTP application logic directly.

Implements the 4 abstract methods from the Node base class:

  1. updateToken()

    • Generates a signed token using jwtSecret.
    • Token is used to authenticate with other servers in the Net.

  2. doFetch(url, options)

    • Performs HTTP requests in a Node.js environment (e.g., using node-fetch, http, or undici).

  3. requestSocket(routerId)

    • Initiates an outbound WebSocket connection to the router with the given ID.
    • Wraps the connection in an EmitterAdapter.

  4. closeSocket(socket)

    • Attempts graceful socket.close(), then forcefully closes the socket via terminate() if it hangs.

Additional Server-Specific Features:

  • Connection Management:

    • Maintains a httpServer for listening to HTTP upgrade requests.
    • Owns a wsServer in "manual upgrade" mode (no automatic path-based handling).
    • Accepts inbound WebSocket connections via HTTP upgrade events.

  • Inbound Event Cues:

    • _connectionCue: handles 'connection' events from accepted WebSocket connections.
    • _upgradeCue: listens for HTTP 'upgrade' events and delegates to handleUpgrade() for WebSocket handling.

  • Connection Policy:

    • Accepts up to inboundConnectivity simultaneous inbound WebSocket connections.
    • You can override handleConnection() or handleUpgrade() to customize acceptance logic or authentication.

Notes:

  • The Server only manages WebSocket communication (inbound/outbound).
  • For HTTP application routes or APIs, plug in frameworks like Express, Fastify, etc., via httpServer.

(protected) handleUpgrade(req, socket, head)

Handles incoming HTTP upgrade requests for WebSocket connections.

This method:

  • Parses the incoming request URL and compares its pathname to the configured Server path derived from this.info.url.
  • Uses this.accepts(req, socket, head) to decide whether to accept the upgrade.
  • If accepted, completes the WebSocket handshake using wsServer.handleUpgrade().
  • Wraps the new WebSocket in an EmitterAdapter and registers it under the user.
  • If the server is currently active (playing), immediately activates the adapter.
  • Emits a 'connection' event with the adapter for listeners.

Note: If the upgrade is not accepted, the socket is left open — you may want to handle socket closure explicitly in that case.

The HTTP upgrade request.

The network socket for the connection.

Initiates an outbound WebSocket connection to a specified server.
